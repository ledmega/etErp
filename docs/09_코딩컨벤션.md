### Java 코드

| 대상 | 규칙 | 예시 |

| **클래스** | PascalCase | `EmployeeService`, `HrController` |

| 대상 | 규칙 | 예시 |

## 2. 클래스별 역할 및 작성 원칙

### Controller
```java
@Controller  // 또는 @RestController (API용)
@RequestMapping("/hr/employees")
@RequiredArgsConstructor
public class EmployeeController {

    private final EmployeeService employeeService;

    // GET: 목록 조회
    @GetMapping
    public String list(@ModelAttribute SearchDto dto, Model model) {
        model.addAttribute("employees", employeeService.findAll(dto));
        return "hr/employee/list";
    }

    // POST: 등록
    @PostMapping
    @ResponseBody
    public ApiResponse<EmployeeResponse> create(
            @Valid @RequestBody EmployeeRequest request) {
        return ApiResponse.success(employeeService.create(request));
    }
}
```

**원칙:**
- 비즈니스 로직 금지 (Service 위임)
- 입력값 `@Valid` 검증 필수
- `@RequiredArgsConstructor` + final 필드 주입

### Service
```java
@Service
@Transactional(readOnly = true)   // 기본 읽기 전용
@RequiredArgsConstructor
public class EmployeeServiceImpl implements EmployeeService {

    private final EmployeeRepository employeeRepository;
    private final EmployeeMapper employeeMapper;

    @Override
    public Page<EmployeeResponse> findAll(SearchDto dto, Pageable pageable) {
        return employeeRepository.findAll(dto, pageable)
                .map(employeeMapper::toResponse);
    }

    @Override
    @Transactional   // 쓰기 작업만 별도 선언
    public EmployeeResponse create(EmployeeRequest request) {
        Employee employee = employeeMapper.toEntity(request);
        return employeeMapper.toResponse(employeeRepository.save(employee));
    }
}
```

**원칙:**
- 인터페이스 + 구현체 분리
- `@Transactional(readOnly = true)` 클래스 레벨 기본
- 쓰기 메서드만 `@Transactional` 별도 선언

### Entity
```java
@Entity
@Table(name = "hr_employee")
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED)
@ToString(exclude = {"dept"})
public class Employee extends BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "emp_id")
    private Long empId;

    @Column(name = "emp_no", nullable = false, unique = true, length = 20)
    private String empNo;

    @Column(name = "emp_name", nullable = false, length = 100)
    private String empName;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "dept_id")
    private Department dept;

    // 비즈니스 메서드
    public void updateInfo(String empName, String email, String phone) {
        this.empName = empName;
        this.email = email;
        this.phone = phone;
    }

    public void resign(LocalDate resignDate) {
        this.resignDate = resignDate;
        this.empStatus = "R";
    }
}
```

**원칙:**
- `@Setter` 사용 금지 → 비즈니스 메서드로 상태 변경
- `fetch = FetchType.LAZY` 기본
- `@NoArgsConstructor(access = AccessLevel.PROTECTED)`

    @NotBlank
    @Size(max = 100)

    @NotNull
    private Long deptId;

    @NotNull
    @PastOrPresent

}

// Response DTO
@Getter

    private Long empId;
    private String empNo;
    private String empName;
    private String deptName;
    private String positionName;
    private LocalDate hireDate;
    private String empStatus;

## 3. 예외 처리 원칙

```java
// 커스텀 예외
@Getter
public class ErpException extends RuntimeException {
    private final ErrorCode errorCode;

    public ErpException(ErrorCode errorCode) {
        super(errorCode.getMessage());
        this.errorCode = errorCode;
    }
}

// 에러 코드 Enum
@Getter
@RequiredArgsConstructor
public enum ErrorCode {
    EMPLOYEE_NOT_FOUND("E001", "직원을 찾을 수 없습니다."),
    DUPLICATE_EMP_NO("E005", "이미 존재하는 사번입니다."),
    INVALID_LEAVE_DATE("E002", "휴가 날짜가 올바르지 않습니다.");

    private final String code;
    private final String message;
}

// 전역 예외 처리
@ExceptionHandler(ErpException.class)
public ResponseEntity<ApiResponse<Void>> handleErpException(ErpException e) {
    return ResponseEntity
        .status(HttpStatus.BAD_REQUEST)
        .body(ApiResponse.fail(e.getErrorCode()));
}
```
