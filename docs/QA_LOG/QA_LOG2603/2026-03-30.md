## Q4. MariaDB 설정 및 DB/계정 생성

**질문:**
> 마리아 DB 쓸건디 `218.237.70.67:33306` 스키마는 아직 안만들었으니 만들어야하고
> 데이터베이스 이름을 `eterp` 로 할까 생각 중임. 계정도 `eterp` 로 생성할까?

**답변 요약:**

### DB/계정 생성 SQL (root 실행)
```sql
CREATE DATABASE IF NOT EXISTS eterp
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'eterp'@'%' IDENTIFIED BY 'eterp1234!';
GRANT ALL PRIVILEGES ON eterp.* TO 'eterp'@'%';
FLUSH PRIVILEGES;
```

### application-dev.yml 설정
```yaml
spring:
  datasource:
    url: jdbc:mariadb://218.237.70.67:33306/eterp?characterEncoding=UTF-8&serverTimezone=Asia/Seoul
    username: eterp
    password: eterp1234!
    driver-class-name: org.mariadb.jdbc.Driver
```

### build.gradle.kts 의존성
```kotlin
runtimeOnly("org.mariadb.jdbc:mariadb-java-client:3.5.1")
```

### DB 접속 정보
| 항목 | 값 |
|------|-----|
| Host | `218.237.70.67` |
| Port | `33306` (기본 3306 아님!) |
| DB명 | `eterp` |
| 계정 | `eterp` |

## Q6. Java Reactive 방식으로 개발 + 한글 주석 요청

**질문:**
> 이 프로젝트는 자바 리액티브를 배우기 위해서 하는 것도 있어서
> 자바 리액티브 방식으로 개발 진행 예정이고
> 공부를 위해서이니까 주석도 한글로 꼼꼼히 달아줘.

**답변 요약:**

### 기존 MVC → WebFlux 전환 변경사항

| 항목 | MVC (변경 전) | WebFlux (변경 후) |
|------|---------------|-------------------|
| 웹 프레임워크 | Spring MVC | Spring WebFlux |
| 서버 | Tomcat | Netty (논블로킹) |
| DB 접근 | Spring Data JPA | Spring Data R2DBC |
| DB 드라이버 | JDBC | R2DBC (r2dbc-mariadb) |
| 반환 타입 | `Employee`, `List<T>` | `Mono<T>`, `Flux<T>` |
| 보안 | Spring Security Servlet | Spring Security Reactive |

### 핵심 개념 정리
| 타입 | 설명 | 기존 비교 |
|------|------|-----------|
| `Mono<T>` | 0~1개 비동기 결과 | `Optional<T>` 비동기 버전 |
| `Flux<T>` | 0~N개 비동기 스트림 | `List<T>` 비동기 버전 |

### 핵심 Operator 요약
| Operator | 용도 |
|----------|------|
| `map()` | 동기 1:1 변환 (Entity → DTO) |
| `flatMap()` | 비동기 연쇄 (Mono → Mono) |
| `filter()` | 조건 필터링 |
| `switchIfEmpty()` | 비어있을 때 대체 (404 처리) |
| `doOnNext()` | 사이드 이펙트 (로그) |
| `zipWith()` | 두 Mono 병렬 실행 후 결합 |
| `collectList()` | Flux → Mono<List> |
| `onErrorResume()` | 에러 처리 |
| `then()` | 결과 버리고 Mono<Void> 반환 |
