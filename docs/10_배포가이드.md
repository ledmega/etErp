## 3. application-prod.yml

```yaml
spring:
  datasource:
    url: ${DB_URL}               # 환경변수로 주입
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    hikari:
      maximum-pool-size: 20
      minimum-idle: 10

  jpa:
    hibernate:
      ddl-auto: validate         # 운영: 스키마 자동 변경 금지
    show-sql: false

  data:
    redis:
      host: ${REDIS_HOST}
      port: ${REDIS_PORT}
      password: ${REDIS_PASSWORD}

  thymeleaf:
    cache: true                  # 운영: 캐시 활성화

logging:
  level:
    com.et.erp: INFO
    org.springframework: WARN

springdoc:
  swagger-ui:
    enabled: false               # 운영: Swagger 비활성화
```

## 4. 서버 실행

### 직접 실행 (jar)

```powershell
# 환경변수 설정 후 실행
$env:DB_URL="jdbc:postgresql://운영DB서버:5432/eternp"
$env:DB_USERNAME="eternp"
$env:DB_PASSWORD="운영DB비밀번호"
$env:REDIS_HOST="redis서버"
$env:REDIS_PORT="6379"
$env:REDIS_PASSWORD="redis비밀번호"
$env:JWT_SECRET="your-super-secret-256bit-key"

java -jar `
  -Dspring.profiles.active=prod `
  -Dfile.encoding=UTF-8 `
  -Xms512m -Xmx1024m `
  build/libs/etErp-1.0.0.jar
```

### Windows 서비스 등록 (NSSM 사용)

```powershell
# NSSM 설치 후
nssm install etERP "java"
nssm set etERP AppParameters "-jar -Dspring.profiles.active=prod E:\apps\etErp\etErp-1.0.0.jar"
nssm set etERP AppDirectory "E:\apps\etErp"
nssm set etERP AppEnvironmentExtra "DB_URL=jdbc:postgresql://..."
nssm start etERP
```

## 5. Nginx 설정 (Reverse Proxy)

```nginx
upstream eternp_backend {
    server 127.0.0.1:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name erp.yourcompany.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name erp.yourcompany.com;

    ssl_certificate     /etc/ssl/certs/erp.crt;
    ssl_certificate_key /etc/ssl/private/erp.key;

    # 보안 헤더
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;

    # 정적 파일 캐싱
    location /static/ {
        proxy_pass http://eternp_backend;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    location / {
        proxy_pass http://eternp_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 타임아웃
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;

        # 업로드 파일 크기
        client_max_body_size 50m;
    }
}
```

## 6. DB 마이그레이션 (Flyway)

스키마 변경은 Flyway를 통해 관리합니다.

```
src/main/resources/db/migration/
├── V1__init_schema.sql           # 최초 스키마 생성
├── V2__add_hr_tables.sql         # HR 테이블 추가
├── V3__add_accounting_tables.sql # 회계 테이블 추가
└── V4__insert_init_data.sql      # 초기 데이터
```

```yaml
# application.yml에 Flyway 설정
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true
```

**규칙:**
- 파일명: `V{버전}__{설명}.sql`
- 버전은 순차적으로 증가
- 한번 배포된 마이그레이션 파일 수정 금지

## 7. 배포 절차 (운영 배포)

```
1. 개발 완료 → Git PR → 코드 리뷰 → main 브랜치 머지

2. 빌드
   ./gradlew clean build -x test

3. 배포 전 DB 백업
   pg_dump -h 운영DB -U eternp eternp > backup_YYYYMMDD.sql

4. Flyway 마이그레이션 확인
   ./gradlew flywayInfo

5. 기존 서비스 중지
   nssm stop etERP  (또는 systemctl stop eternp)

6. JAR 파일 교체
   copy build\libs\etErp-1.0.0.jar E:\apps\etErp\

7. 서비스 시작
   nssm start etERP

8. 헬스체크
   curl http://localhost:8080/actuator/health

9. 운영 로그 확인
   Get-Content E:\logs\etErp\application.log -Wait -Tail 100
```

## 8. 모니터링

### Spring Actuator 엔드포인트

| 엔드포인트 | 설명 | 노출 |
|-----------|------|------|
| `/actuator/health` | 서버 상태 | 운영 허용 |
| `/actuator/info` | 앱 정보 | 운영 허용 |
| `/actuator/metrics` | 메트릭 | 내부망만 |
| `/actuator/loggers` | 로그 레벨 | 내부망만 |

```yaml
# 운영 환경 actuator 설정
management:
  endpoints:
    web:
      exposure:
        include: health, info
  endpoint:
    health:
      show-details: when-authorized
```

### 로그 위치
```
E:\logs\etErp\
├── application.log         # 메인 로그
├── error.log               # 에러 로그
└── audit.log               # 감사 로그
```

## 9. 롤백 절차

```
1. 문제 발생 → 즉시 서비스 중지

2. 이전 JAR로 교체
   copy E:\apps\etErp\backup\etErp-이전버전.jar E:\apps\etErp\etErp-1.0.0.jar

3. DB 롤백 (필요시)
   psql -h 운영DB -U eternp eternp < backup_YYYYMMDD.sql

4. 서비스 재시작
   nssm start etERP

5. 원인 분석 후 re-deploy
```
