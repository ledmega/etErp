## 2. 프로젝트 배경 및 목적

현재 사내 업무 처리가 Excel, 개별 시스템, 수작업 등으로 분산되어 있어
**데이터 일관성 확보**와 **업무 효율화**를 위해 통합 ERP 시스템을 구축합니다.

### 현황 문제점
- 부서별 개별 Excel 관리로 인한 데이터 중복 및 오류
- 실시간 재고/매출 현황 파악 불가
- 결재 및 승인 프로세스 수작업 처리
- 인사/급여 데이터 연계 미흡

### 기대 효과
- 전사 데이터 통합으로 실시간 경영 현황 파악
- 업무 프로세스 자동화로 생산성 향상
- 정확한 재고 관리 및 원가 분석
- 내부 통제 강화 및 감사 추적성 확보

## 3. 주요 기능 (모듈) 목록

### Phase 1 (1차 오픈 목표)
| 모듈 | 기능 | 우선순위 |
|------|------|----------|
| **시스템 관리** | 사용자/권한/메뉴 관리 | ⭐ 최우선 |
| **인사관리** | 직원 정보, 부서, 직급 관리 | ⭐ 최우선 |
| **근태관리** | 출퇴근, 휴가, 초과근무 | 🔴 높음 |
| **재무/회계** | 전표, 계정과목, 예산 | 🔴 높음 |
| **구매관리** | 거래처, 발주, 입고 | 🟡 중간 |

### Phase 2 (2차 오픈 목표)
| 모듈 | 기능 | 우선순위 |
|------|------|----------|
| **영업관리** | 고객, 수주, 매출 | 🟡 중간 |
| **재고관리** | 품목, 창고, 이동 | 🟡 중간 |
| **급여관리** | 급여 계산, 명세서 | 🟡 중간 |

## 4. 프로젝트 범위

### In Scope (포함)
- 웹 기반 사내 ERP 시스템
- PC 브라우저 지원 (Chrome, Edge)
- 사내망 내부 서비스
- 역할 기반 접근 제어 (RBAC)
- 데이터 감사 로그

### Out of Scope (제외)
- 모바일 앱 (향후 검토)
- 외부 연계 API (세금계산서 연동 등 2차 검토)
- EDI 연동
